报错

MISCONF Redis is configured to save RDB snapshots, but it is currently not able to persist on disk. Commands that may modify the data set are disabled, because this instance is configured to report errors during writes if RDB snapshotting fails (stop-writes-on-bgsave-error option). Please check the Redis logs for details about the RDB error.

原因：
因为强制把redis快照关闭了导致不能持久化的问题

解决方法：
修改redis.conf中的stop-writes-on-bgsave-error为no

奇奇怪怪的知识点

redis事务

参考https://www.jianshu.com/p/c9f5718e58f0

redis开始事务之后，如果对redis操作完了，只会解绑连接，不会做close的操作，因为redis并不知道事务是否已经执行完。

在spring中要使用Redis注解式事务，首先要设置RedisTemplate的enableTransactionSupport属性为true，然后配置spring的事务，所有使用这个template的redis操作都必须走注解式事务，要不然会导致连接一直占用，不关闭。

但是通常情况下，redis一般只做缓存，对数据要求不会很高，所以比较少开启事务，一般只是setter和getter。如果要求一致性很高，建议使用数据库的唯一索引做。
