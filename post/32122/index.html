火焰图简述

火焰图（Flame Graph）是由 Linux 性能优化大师 Brendan Gregg 发明的，和所有其他的 profiling 方法不同的是，火焰图以一个全局的视野来看待时间分布，它从底部往顶部，列出所有可能导致性能瓶颈的调用栈。

步骤：

  1. 运行代码，直接访问http://localhost:8080/debug/pprof/,即可查到查看程序运行相关信息。
  2. export PPROF_TMPDIR="pprof/"指定pprof文件保存的位置，不设置则保存默认位置，不便于后续维护开发
  3. go tool pprof -seconds 10 http://127.0.0.1:8080/debug/pprof/profile采集程序运行10s中的CPU信息，并生成pprof文件，文件名大致如pprof.samples.cpu.001.pb.gz
  4. go tool pprof -http=:8081 ~/pprof/pprof.samples.cpu.001.pb.gz生成火焰图

得到的火焰图如下：

  • 默认是图状火焰图：
    可以在图中看到每个方法的耗时和占比
    比如doSomething耗时0.5s，耗时占比3.00%，LocakTz耗时6.64s，耗时占比39.81%，所以如果要优化的话就是优化LocakTz方法

  • 点击VIEW中可以切换视图，如Falme Graph，可以看到如下的图，可以从另外一个角度看到每个方法的耗时比例

  • 点击VIEW中可以切换视图，如Source，可以看到如下图，从源码角度看到每个方法的耗时和比例

  • 点击VIEW中可以切换视图，TOP中的含义这里做一下解释

    • flat/flat%：分别表示在当前层级 CPU 的占用时间和百分比。
    • cum/cum%：分别表示截止到当前层级累积的 CPU 时间和占比。
    • sum%：所有层级的 CPU ，从小到大一直累积到100%
    • name：对应数据的方法路径

TIPS:

  • 在benchmark中生成火焰图，参数如下：
    • cpuprofile=文件名
    • memprofile=文件名
    • memprofilerate=N 调整记录速率为原来的 1/N。
    • blockprofile=文件名
      示例：
      go test -bench . -cpuprofile=cpu.prof
      go test -bench . -memprofile=./mem.prof
  • 火焰图一般结合wrk压测组件一起使用，不压测怎么知道接口可用性，使用方法自行百度，比较简单
